# 示例项目A教程

用于演示项目实际教程的推荐写法，后续新增项目时可直接复制本文件改写。

## 一、项目概况

- 项目类型：`剧情短片`
- 适用场景：`新人培训 / 项目复盘`
- 核心工具：`Lib TV`、`即梦`、`可灵 O3`
- 交付目标：完成可交付的竖屏短片，并沉淀可复用方法

## 二、项目目标

- 明确项目想解决的问题，而不是只罗列工具步骤
- 优先统一分镜逻辑、角色设定和场景风格
- 减少生视频阶段的连锁返工

## 三、制作拆解

### 1. 文字剧本 / 分镜定调
- 先确认叙事成立、镜头逻辑清晰
- 提前确定角色一致性、场景统一性和情绪基调
- 高风险镜头单独标记，避免后续集中返工

### 2. 生图阶段
- 先完成角色设定图和场景设定图
- 再按镜头拆分生成关键帧
- 生图完成后统一审核，不通过不进入生视频

### 3. 生视频阶段
- 先验证重点镜头，再推进批量生成
- 问题先判断属于分镜、生图还是生视频执行本身
- 镜头连贯性问题优先回到前置阶段修正

### 4. 配音 / 剪辑 / 导出
- 配音内容与分镜脚本保持一致
- 剪辑阶段重点做整合和交付完善，不承担前置方向返工
- 导出前统一检查版本、命名和归档完整性

## 四、关键问题复盘

### 1. 本项目最容易返工的点
- 镜头衔接
- 角色一致性
- 音画节奏不匹配

### 2. 本次如何规避
- 把审核前移到分镜和生图阶段
- 生图完成后先做统一审核，再进入生视频
- 剪辑阶段只处理交付层问题，不兜底前序方向问题

## 五、最终交付

- 分镜脚本
- 原画关键帧 + 源文件
- 视频 / 动画素材 + 工程文件
- 成片

- 哪些镜头写法可复用
- 哪些模型组合最稳定
- 哪些问题必须前置解决
